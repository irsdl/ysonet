# Testing (ysonet.Tests)

- 2026-07-21 - New tests that write any file (fixture, input, payload, or a dropped marker) must write to the first writable directory in this order: workspace-root `temp` (gitignored) -> `Path.GetTempPath()` -> `C:\Windows\Temp` (via `%SystemRoot%`) -> `C:\temp`, verifying the file still exists after writing and falling through if it vanished. Use the `WriteTestArtifact` / `ResolveTestArtifactDir` / `FindWorkspaceRoot` helpers; never hardcode a machine path. - Antivirus sometimes deletes a generated payload or dropped marker as a false positive right after it is written, causing flaky failures; an AV-excluded workspace-root `temp` is the most reliable spot. Never loosen a behavioral assertion to dodge AV (test-integrity policy).
- 2026-07-22 - Antivirus can intercept `csc.exe` and make the compile-based gadget tests (the ones that compile a `.cs`: `DataSetOldBehaviourFromFile`, `ActivitySurrogateSelectorFromFile`, `XamlAssemblyLoadFromFile`, `EveryGadgetGeneratesAPayload`, and the `*FromFile` fire tests) hang with low CPU, so the run never reaches the later tests. Pause AV or exclude the build output dir; killing `csc` does not unblock a wedged compile. - Real-time AV scanning of the temp-compiled assembly deadlocks the legacy CodeDom (`CSharpCodeProvider`) compile.
