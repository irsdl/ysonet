# Interactive UI (ysonet/Interactive/)

- 2026-07-22 - Screen-redraw convention: a screen must call `ConsoleCursor.ClearScreen()` once when it is entered or re-entered, then redraw in place with `MoveUp` for navigation within that screen; a sub-screen (e.g. an axis checklist) clears on its own entry and the parent clears again on return, wiping it - never append a screen beneath the previous one. - `MoveUp` only redraws in place within a single screen, so a parent -> child -> parent transition without a clear leaves both drawn and the menu appears twice on a real console (the "menu repeats down the screen" stacking bug; fixed in `CategoryFilter`; canonical note on the `Menu` class comment; full detail in `docs/ARCHITECTURE.md` section 4.1).
- 2026-07-22 - A redirected-console test cannot catch a real-console redraw/stacking bug; write the regression test against the `VirtualTerminal` harness (real cursor control) and assert the screen title never appears on two rows of one captured frame - see `CategoryFilterDoesNotStack`. - Under a redirected console `ClearScreen`/`MoveUp` are no-ops and output just appends, so the stacking is invisible to append-mode tests.
- 2026-07-22 - The `Picker` block now sizes to the window: `Picker.Render` calls `FitSizes` each frame, shrinking the preview to 0 then the list toward 1 row so the whole block (`Overhead` = title+help+search+count) fits in `ConsoleCursor.Height()` (from `Term.WindowHeight`). On a short terminal the fixed ~24-line block used to overflow, so the relative `MoveUp` clamped at row 0 and the title scrolled off / the menu stacked (reported "does not work well when the screen is small"). `Height()` returns 0 when we do not control a real console (redirected/tests), which keeps the full 12+8 block and just appends - so redirected tests are unaffected. Regression: `PickerFitsShortWindow` drives the picker in an 80x12 `VirtualTerminal` and asserts the title, Search line and count line all stay on one screen. Same pattern as `ModuleEditor`'s `FrameHeight`/`ColumnsFit` guard. - A fixed-height redraw block must fit the window or the in-place `MoveUp` desyncs on a small screen.
- 2026-07-22 - The single-panel FALLBACK path (`ModuleEditor.RunFallback` -> `EditForm` -> `EditField`, used on a short/narrow real console as well as redirected output) was missing the clear-on-entry convention that the columns path has, so on a small window screens stacked: the top menu stayed above the module picker, and the module-picker info preview (Formatters/Labels/Credit) plus edit-screen leftovers stayed above the settings form (reported "residuals when I go inside the menu"). Fix: `RunFallback` clears at the top of its loop; `EditForm` clears at the top of its loop (covers the module-preview residual on entry and the edit residual on re-entry); `EditField` clears on entry; action rows (Generate/Show command) clear before running, then `PauseForReview()` (only when `ConsoleCursor.CanControl()`) so the payload/command stays readable before the loop clears. Gating the pause on `CanControl` keeps redirected/tests unchanged (no extra keypress, clears are no-ops there). Regression via a new `DriveFallbackFrames` harness (VirtualTerminal + `ModuleEditor.ForceFallback=true` + real cursor control): `FallbackClearsTopMenuOnShortWindow`, `FallbackFormClearsModulePreview`, `FallbackFormClearsEditResidual`. - Every screen transition on a real console must clear or the previous screen shows as a residual/stack; the columns path already did this, the fallback did not.
- 2026-07-22 - The `Picker` window shows up to 12 rows (fewer on a short window, see the FitSizes note above), so an action/option row appended at the BOTTOM of a long list (e.g. the gadget list's `[ Filter by category... ]`) is off-screen until the user types or scrolls. Make it discoverable another way: a hint in the picker title, a footer/legend key (the live columns add `Ctrl+F`), and rely on type-to-filter surfacing it (typing "filter" narrows to that row). - Bottom placement keeps the first real item as the default selection (good for the common Enter-Enter pick), but trades away default visibility of trailing rows.
